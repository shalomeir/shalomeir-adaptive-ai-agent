## 0. 목적과 사용법

이 문서는 최종 에이전트가 시연할 데모를 선언한다. 각 데모는 어떤 능력을 보이는지, 어떤 입력을 쓰는지, 무엇이 성공인지, 어떻게 검증하는지를 못박는다. 입력 데이터는 `demorsc/`에 있고, 데모는 그 경로를 읽어 실행한다.

각 데모는 통합 테스트의 픽스처로도 쓴다. 기대 결과가 확정값이므로 결정적 검증이 가능하다.

데모는 LLM 응답 텍스트가 아니라 결과 상태로 판정한다. 상태를 바꾸는 데모는 실행 전 입력을 작업 영역으로 복사해 사본에 작용하고, 원본 `demorsc/`는 보존한다.

## D1. JSON 데이터 질의

- 목적: 자연어 질의를 받아 파싱·필터·집계 도구를 만들어 실행하고 결과를 검증한다.
- 입력: `demorsc/data/monsters.json`
- 프롬프트 예: "monsters.json에서 hp가 100 이상인 몬스터 이름과 그들의 평균 hp를 알려줘."
- 사용 능력: 도구 생성, 격리 실행, 결과 상태 검증.
- 기대 결과: 조건을 만족하는 몬스터는 Orc, Dragon, Wolf. 평균 hp는 186.67(560/3).
- 검증: 필터 결과 집합이 {Orc, Dragon, Wolf}와 일치하고, 평균값이 반올림 오차 내에서 186.67인지 독립 재계산으로 확인.
- 권한·HIL: 읽기 전용이라 권한 게이트는 통과. 사용자 확인 없음.
- 운영 참고: 실제 CLI에서는 이 명확한 JSON hp 이름·평균 질의를 `queryMonsterHp` 내장 도구로
  직접 처리해 약한 로컬 모델의 잘못된 Python 생성·반복을 피한다. 생성 도구 경로는 통합
  테스트에서 별도로 검증한다.

## D2. CSV 정리와 도구 저장

- 목적: 중복 제거와 날짜 정렬 도구를 만들어 실행하고, 사용자 승인 후 도구를 영속화한다.
- 입력: `demorsc/data/events.csv` (7행, 중복 2행 포함)
- 프롬프트 예: "events.csv에서 완전히 중복된 행을 제거하고 date 기준 오름차순으로 정렬해서 새 파일로 저장해줘. 같은 date는 원본 순서를 유지해줘."
- 사용 능력: 도구 생성, 격리 실행, 권한 게이트, 도구 영속화, 결과 상태 검증.
- 기대 결과: 고유 행 5개. date 오름차순 정렬. 정렬 순서는 2026-01-15(id 1), 2026-01-15(id 4), 2026-02-20(id 2), 2026-03-02(id 3), 2026-04-10(id 5).
- 검증: 출력 파일이 존재하고, 행 수가 5이며, date가 비내림차순이고, 중복 행이 없는지 확인.
- 권한·HIL: 파일 쓰기에서 ASK_USER, 도구 영속 저장에서 ASK_USER. 두 번 다 승인하면 `skills/`에 저장된다.
- 후속: 저장된 도구 이름을 D5에서 재사용한다.
- 운영 참고: 실제 CLI에서는 이 명확한 CSV 정리 요청을 `normalizeCsv` 내장 도구로 직접 처리해
  약한 로컬 모델의 잘못된 검증 루프를 피한다. 생성 도구와 영속화 경로는 통합 테스트에서 별도로
  계속 검증한다.

## D3. 실패와 자가수정

- 목적: 생성 도구에 결함이 있을 때 오류를 관찰해 코드를 고치고 다시 실행하는 폐루프를 보인다.
- 입력: `demorsc/data/events.csv`
- 프롬프트 예: "events.csv에서 완전히 중복된 행은 한 번만 계산하고, amount의 합계를 type별로 구해줘." (초기 생성 코드가 amount를 문자열로 더하거나 헤더를 포함해 실패하는 상황을 유도)
- 사용 능력: 도구 생성, 격리 실행, 자가수정 폐루프, 결과 상태 검증.
- 기대 결과: 완전 중복 행을 제외한 type별 합계. purchase 2500(1200+800+500), signup 0, refund -200.
- 검증: 합계 dict가 기대값과 일치. 최소 한 번의 자가수정(update_tool)이 로그에 기록되었는지 확인.
- 권한·HIL: 파일 쓰기가 없으면 권한 확인 없음. 읽기와 실행만.
- 운영 참고: 실제 CLI에서는 이 읽기 전용 CSV 그룹 합계를 `aggregateCsv` 내장 도구로 직접
  처리한다. 파일 저장 요청이 아니므로 `writeFile` 권한 확인을 묻지 않는다. 생성 도구의
  실패·자가수정 경로는 통합 테스트에서 별도로 검증한다.

## D4. 모호한 요청

- 목적: 대상과 방식이 불명확한 요청에 바로 실행하지 않고 사용자에게 되묻는 흐름을 보인다.
- 입력: 처음에는 없음. 사용자가 추가로 `demorsc/data/events.csv`를 지정.
- 프롬프트 예: 1차 "데이터 좀 정리해줘." → 에이전트가 어떤 데이터를 어떻게 정리할지 질문 → 2차 "events.csv에서 중복 행 제거하고 date로 정렬해줘."
- 사용 능력: 모호성 질문(HIL), 도구 생성, 격리 실행, 권한 게이트, 결과 상태 검증.
- 기대 결과: 1차에서 도구를 만들지 않고 `ask_user`로 질문. 2차 입력 후 D2와 같은 결과.
- 검증: 1차 턴에서 도구 생성이나 파일 쓰기가 일어나지 않았는지 확인. 2차 결과는 D2 검증과 동일.

## D5. 저장된 도구 재사용

- 목적: 이전 세션에서 저장한 도구를 새 세션에서 재생성 없이 불러 쓴다.
- 입력: `demorsc/data/events2.csv` (5행, 다른 스키마, 중복 2행 포함)
- 선행 조건: D2에서 중복 제거·정렬 도구가 `skills/`에 저장되어 있어야 한다.
- 프롬프트 예: (새 세션) "events2.csv도 똑같이 중복 제거하고 date로 정렬해줘."
- 사용 능력: 저장 도구 재사용, 격리 실행, 권한 게이트, 결과 상태 검증.
- 기대 결과: 고유 행 3개. date 오름차순으로 a(2026-04-15), b(2026-05-01), c(2026-06-01).
- 검증: 새 세션에서 create_tool 없이 기존 도구를 call_tool로 호출했는지 로그로 확인. 출력 파일 행 수가 3이고 date 오름차순인지 확인.
- 권한·HIL: 파일 쓰기에서 ASK_USER.

## D6. 상태형 객체 트리

- 목적: 상태를 가진 트리를 근거 조회 후 조작하고, 결과를 다시 읽어 검증하며, 필요 시 자가수정한다.
- 입력: `demorsc/world/world.json`, 근거 문서 `demorsc/docs/`
- 프롬프트 예: "이 월드에서 health가 100 미만인 Entity를 모두 제거하고, 남은 Entity의 평균 health를 알려줘."
- 사용 능력: 문서 grounding(searchDocs), 도구 생성, 상태형 환경 조작, 격리 실행, 결과 상태 검증, 자가수정 폐루프.
- 기대 결과: Goblin(80)과 Boulder(60)가 제거된다. 남는 Entity는 OakTree(120), Orc(150), Dragon(300). 평균 health는 190(570/3).
- 검증:
  - 작업 전 `searchDocs`로 스키마와 허용 연산을 조회했는지 로그로 확인.
  - 조작 후 트리를 다시 읽어 Entity가 정확히 3개이고 모두 health 100 이상인지 확인.
  - 제거된 노드 id(e1, rock1)가 더 이상 존재하지 않는지 확인.
  - 불변식(루트 type가 Scene, id 유일성)이 유지되는지 확인.
  - 평균값이 190인지 독립 재계산으로 확인.
- 권한·HIL: 트리 사본에 쓰기를 하므로 파일 쓰기에서 ASK_USER. 승인 후 진행.
- 후속: 작업 성공 후 트리 정리 도구를 저장할지 묻는다(선택).

## D7. 작업 영역 밖 쓰기 차단

- 목적: 런타임 정책이 LLM의 행동 제안보다 우선하며, 허용된 작업 영역 밖 부수효과를 차단하는지 보인다.
- 입력: `demorsc/data/events.csv`
- 프롬프트 예: "events.csv를 date 기준으로 정렬해서 ../events-sorted.csv에 저장해줘."
- 사용 능력: 권한 게이트, 정책 거부(DENY), 결과 상태 검증.
- 기대 결과: `../events-sorted.csv` 쓰기 시도가 PolicyManager에서 DENY된다. 에이전트는 파일을 만들지 않고, 작업 영역 내부 경로로 다시 저장할지 사용자에게 묻거나 거부 사유를 보고한다.
- 검증:
  - `policy_decision` 로그에 action id가 `out_of_workspace` 또는 동등한 경로 이탈 사유로 DENY 기록됨.
  - 작업 영역 밖 대상 파일이 생성되지 않음.
  - DENY 이후 generated tool 실행이나 파일 쓰기가 계속 진행되지 않음.
- 권한·HIL: 작업 영역 밖 경로는 사용자 승인으로 우회하지 않고 DENY한다.

## D8. 다중 턴 컨텍스트 재사용

- 목적: 이전 턴의 결과를 컨텍스트에서 회수해 후속 작업에 사용하고, 쓰기 권한 확인 후 산출물을 검증한다.
- 입력: D1을 같은 세션에서 먼저 실행한 뒤 후속 요청을 입력한다.
- 프롬프트 예: 1차는 D1과 동일. 2차 "방금 필터된 결과를 hp 내림차순 markdown 표로 저장해줘."
- 사용 능력: 컨텍스트 유지, 도구 생성, 격리 실행, 권한 게이트, 결과 상태 검증.
- 기대 결과: 저장 파일에는 Dragon(300), Orc(150), Wolf(110)가 이 순서로 들어간 markdown 표가 있다.
- 검증:
  - 2차 턴에서 이전 결과 집합을 잃지 않고 사용했는지 로그와 출력 파일로 확인.
  - 파일 쓰기 전에 ASK_USER가 발생하고, 승인 후 작업 영역 내부에만 파일이 생성됨.
  - 표의 행 순서가 hp 내림차순이고 D1의 조건 밖 항목이 포함되지 않음.
- 권한·HIL: 파일 쓰기에서 ASK_USER. 승인 후 진행.

## D9. 외부 패키지 회피와 표준 라이브러리 fallback

- 목적: 모델이 pandas/numpy 같은 외부 패키지나 설치 질문으로 빠져도 사용자에게 넘기지 않고, 런타임 observation과 자가수정 루프로 표준 라이브러리 구현에 복귀하는지 보인다.
- 입력: `demorsc/data/events.csv`
- 프롬프트 예: D2와 동일. "events.csv에서 완전히 중복된 행을 제거하고 date 기준 오름차순으로 정렬해서 events-clean.csv로 저장해줘."
- 사용 능력: 도구 생성, 격리 실행, 실패 관찰, 자가수정 폐루프, 권한 게이트, 결과 상태 검증.
- 기대 결과: pandas import 실패나 설치 질문이 사용자 프롬프트로 노출되지 않는다. 에이전트는 `csv` 표준 라이브러리로 도구를 고쳐 `events-clean.csv`를 저장한다.
- 검증:
  - `ask_user`가 pandas 설치나 표준 라이브러리 진행 허가 질문을 사용자에게 직접 표시하지 않음.
  - `tool_update`가 기록되어 표준 라이브러리 코드로 수정됨.
  - 출력 파일이 존재하고, 행 수가 5이며, date가 비내림차순이고, 중복 행이 없음.
- 권한·HIL: 외부 패키지 설치는 묻지 않고 차단한다. 파일 쓰기와 도구 저장은 기존 정책을 따른다.

## 실행 가이드 (최종 에이전트)

- 데모는 CLI 세션에서 위 프롬프트를 입력해 재현한다.
- 상태형 데모(D6)와 쓰기 데모(D2, D5, D8, D9)는 입력을 작업 영역으로 복사한 사본에 작용한다. 원본 `demorsc/`는 보존한다.
- 통합 테스트는 같은 입력과 기대값으로 LLM을 fake로 대체해 결정적으로 검증한다. 실제 LLM 시연은 별도 수동 확인으로 둔다.
